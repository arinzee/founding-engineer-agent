# Python Tutor Platform — Initiative and Epic Map

## Initiative 1: Learner Experience
Goal: create a child-safe tutoring experience that is the only learner-facing interaction surface.

### Epics
- Learner access and session entry
- Child-safe tutoring interface
- Lesson and exercise interaction
- Learner progress visibility

## Initiative 2: Parent Experience
Goal: give parents insight, oversight, and communication without exposing internal system complexity.

### Epics
- Parent dashboard
- Parent-agent interaction
- WhatsApp parent channel
- Parent controls and preferences

## Initiative 3: Agent Orchestration
Goal: define and coordinate the learner, parent, curriculum, and super-agent layers.

### Epics
- Learner agent runtime
- Parent agent runtime
- Curriculum agent integration
- Super-agent supervision
- Agent interaction and context routing

## Initiative 4: Safety, Permissions, and Boundaries
Goal: enforce child-safe and role-safe behavior at the system level.

### Epics
- Role and identity model
- Data visibility model
- Action authority model
- AI context eligibility rules
- Forbidden path enforcement

## Initiative 5: Progress and Learning Intelligence
Goal: track and surface the learner’s progress in ways useful to child, parent, and operator.

### Epics
- Learning state and history
- Parent progress insights
- Operator / super-agent progress intelligence
- Risk and escalation signals

## Initiative 6: Client Provisioning and Customization
Goal: support a per-client/family tutor experience that can be tailored safely.

### Epics
- Client/family onboarding
- Super-agent configuration
- Client-specific tutor preferences
- Tenant/client isolation

## Initiative 7: Deployment and Operations
Goal: deploy and operate the system reliably on a VPS in containers.

### Epics
- VPS/container deployment baseline
- Auth, secrets, and config management
- Audit logging and observability
- Backup, rollback, and recovery
