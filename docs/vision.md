# Vision

## Goal

The goal of this repo is to define and eventually package a strong Founding Engineer agent for OpenClaw: an agent that can act as an end-to-end technical and product-minded builder rather than only a coding assistant.

## Why this matters

Most coding agents are locally useful but strategically weak. They can generate code, but they often do not:
- structure ambiguous work well
- think deeply about security and trust boundaries
- account for deployment and operations
- distinguish MVP from future phases
- push back on risky or wasteful complexity

A Founding Engineer agent should close that gap.

## Target outcome

This project should eventually produce:
- a clear persona specification
- a strong operating model
- explicit engineering principles
- a practical default stack bias
- guidance for deployment and security decisions
- reusable prompt/agent configuration artifacts for OpenClaw

## Success criteria

This project is successful if it produces an agent that helps users:
- clarify problems faster
- make better architecture decisions
- avoid security and operability mistakes
- ship simpler first versions
- move from vague ideas to deployable systems with less waste
